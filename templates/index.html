Architecture

How the Connection Works

Three layers working together — browser, Flask bridge, and Python ML model — all communicating through JSON over HTTP.

🌐
Browser
HTML + CSS + JS
You type a command. JavaScript sends it to Flask via fetch() POST request.
fetch("/chat")
⚗️
Flask
app.py — Bridge
Receives the command, passes it to the ML model, returns JSON to the browser.
model.predict()
🧠
ML Model
TF-IDF + Logistic Reg
Classifies intent from text. Returns intent label + confidence score.
// script.js — browser sends command to Flask
const response = await fetch("/chat", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ message: "report defect on line 3" })
});
const data = await response.json();
// data = { intent: "report_defect", confidence: 73.0, response: "..." }
showMessage(data.response, data.intent, data.confidence);
# app.py — Flask receives and routes to ML
@app.route("/chat", methods=["POST"])
def chat():
    data    = request.get_json()
    message = data.get("message", "").lower()

    intent, confidence = model.predict(message)
    response           = model.get_response(intent, message)

    return jsonify({
        "intent":     intent,      # "report_defect"
        "confidence": confidence,  # 73.0
        "response":   response     # "Defect saved! ID #4821..."
    })
# ML model — TF-IDF + Logistic Regression
def predict(self, command):
    intent     = self.pipeline.predict([command])[0]
    proba      = self.pipeline.predict_proba([command])[0]
    confidence = max(proba)

    # Normalize: removes bias from number of classes
    n          = len(self.pipeline.classes_)
    normalized = (confidence - 1/n) / (1 - 1/n)
    return intent, round(normalized * 100, 1)
    # → ("report_defect", 73.0)
